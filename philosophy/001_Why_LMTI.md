# 001 Why LMTI

I keep returning to one small question.

Why does AI need to rediscover the same project again and again?

A human engineer does not reread the entire codebase every morning.

At first, yes, we search. We open files. We get lost.

But after a while, something changes.

We begin to carry a model of the system in our head.

We remember which modules matter.

We remember where risk usually hides.

We remember which rules are written in code, and which rules only exist because
someone once made a painful mistake.

Current AI systems are very good at reasoning inside a window of context.

But that window is temporary.

The work of understanding often disappears after the task ends.

LMTI exists to ask whether that has to be true.

What if a project could be compiled into a local mind file?

What if an agent could start with structured understanding instead of wandering
through the same files each time?

What if intelligence is not only a property of the model, but also a property of
how understanding is organized?

This is the beginning of LMTI.

Not certainty.

Not a claim.

An experiment.
