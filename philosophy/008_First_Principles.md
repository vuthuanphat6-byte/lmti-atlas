# 008 First Principles

These are the first principles I want LMTI to remember.

They may change.

But they should not change casually.

## 1. Understanding should persist

If an agent understands something useful about a project, that understanding
should not disappear immediately after the task.

## 2. Memory is not enough

Storing text is not the same as organizing meaning.

LMTI should preserve structure, not just data.

## 3. Local first

The project mind should live locally by default.

Private project knowledge should not need a cloud service to become useful.

## 4. Models are replaceable

LMTI should not depend on one model, one vendor or one API.

The mind layer should stay independent.

## 5. Privacy is part of cognition

Sensitive knowledge is not ordinary data.

If a mind cannot protect what it knows, it is not safe enough to trust.

## 6. Claims need evidence

If we say LMTI reduces exploration, we measure exploration.

If we say LMTI improves context, we compare context.

If we cannot measure something yet, we say that clearly.

## 7. Small experiments beat large promises

The project should grow through experiments that teach us something.

Each sprint should make the questions sharper.

## 8. Philosophy can overrule implementation

Code is flexible.

The reason for the code is harder to protect.

If implementation and philosophy conflict, the philosophy wins.
