# LMTI Philosophy

This directory does not describe implementation.

It describes why LMTI exists.

Every contributor should read these notes before contributing code.

They are written as founder notes, not product documentation.

The underscore-numbered files below are the canonical reading path.

Older draft notes with spaced filenames are kept as historical fragments.

## Rule

If source code conflicts with philosophy, the code must be questioned.

If an implementation is clever but weakens cognition, reject it.

Code is only one part of LMTI.

The philosophy is equally important.
