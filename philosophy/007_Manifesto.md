# 007 Manifesto

To future contributors,

I started LMTI because I could not stop thinking about one problem.

AI systems can reason beautifully for a moment, then lose the shape of what they
just understood.

They can inspect a project, find useful structure, explain it clearly, and then
the next task still begins with searching again.

That feels incomplete to me.

Not broken.

Incomplete.

LMTI exists to explore that missing layer.

Can understanding persist?

Can a project have a local mind file?

Can an agent use that mind file to think with less wandering?

Can we make AI coding work feel less like repeated discovery and more like
growing familiarity?

I do not want this project to become a place for big claims.

I want it to become a place for careful questions.

Open source matters here because the idea needs pressure.

It needs people who disagree.

It needs people who test assumptions.

It needs people who care enough to say, "This part is not proven."

Please do not treat the code as the whole project.

The code is only one expression of the idea.

The philosophy matters too.

The benchmarks matter too.

The failures matter too.

If an implementation looks clever but weakens the purpose, question it.

If a benchmark shows the idea is weaker than we hoped, keep the benchmark.

Evidence is more important than belief.

I would rather LMTI become a small honest research project than a large system
built on claims we never measured.

So contribute with curiosity.

Be skeptical.

Protect simplicity.

Protect privacy.

Protect the questions.

We are not trying to prove LMTI is right.

We are trying to discover whether a better Mind can create a better AI.

Edgar Vu
