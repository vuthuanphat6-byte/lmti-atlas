# 006 Research Principles

LMTI should be built like research before it is built like product.

This means we have to be careful with belief.

Belief is useful at the beginning.

It gives direction.

But belief is not evidence.

If LMTI claims to reduce unnecessary exploration, we should measure exploration.

If LMTI claims to improve context, we should compare context quality.

If LMTI claims to protect knowledge, we should test leakage paths.

Every claim should become a question that can be checked.

This project should prefer small experiments over large promises.

Local first.

Measurable first.

Replaceable parts.

No hidden cloud.

No dependency on one model.

No magic that cannot be inspected.

If a benchmark proves us wrong, that is not failure.

It is information.

The worst outcome is not being wrong.

The worst outcome is building a beautiful system that no one can honestly
evaluate.

LMTI should stay close to evidence.

The code should make experiments easier.

The experiments should make the philosophy sharper.

And when philosophy and implementation disagree, we should stop and think
before adding more code.
