# Naming Clarification

> Status: Local-alpha documentation

LMTI is the product.

LMTI means Long Memory / Project Intelligence Layer.

ATLAS is an internal/legacy codename and may still appear in research archives,
old sprint documents, local-alpha package names, or the root workspace name.
Those references are intentional historical/internal naming, not the public
product identity.

The private root package name `atlas-root` is local-alpha workspace naming. It
does not define the public product identity.

The publish repository is `vuthuanphat6-byte/lmti-atlas`. Do not use a private
or legacy `/atlas` repository path as the public identity.

Project Atlas refers to the compiled project map generated by `lmti compile`.
It is an artifact name, not a separate product.

All `@atlas/*` packages are internal implementation packages of LMTI during
local alpha. Agents should not treat them as a separate framework or product.

The primary CLI command is `lmti`. If an `atlas` bin or command alias is still
available, treat it as a legacy/internal alias and do not use it as the primary
documented command.

Identity summary:

```text
LMTI is the product.
ATLAS is an internal/legacy codename and Project Atlas artifact name.
Publish repo is vuthuanphat6-byte/lmti-atlas.
When installed in a host repository, LMTI is not part of the host application runtime.
LMTI is connected by context, not by runtime ownership.
```
